Was wir vorbereitet haben
--------------------------
- FastAPI Backend (requesthandler.py) mit /api/chat Endpunkt

  und schickt sie zusammen mit der Frage an das Azure OpenAI LLM
- Azure AI Foundry Integration ueber den openai-Python-Client (AzureOpenAI)
- Chat-Frontend (webpage/) mit Dark Mode, Spracheingabe (STT) und Vorlesen (TTS)
- Dockerfile fuer HuggingFace Docker Space (Port 7860)
- requirements.txt mit allen Python-Abhaengigkeiten


Was du noch erledigen musst, damit es laeuft
---------------------------------------------

1. Azure AI Foundry Key eintragen
   - Geh in deinen HuggingFace Space unter Settings > Secrets
   - Trage folgende vier Werte ein (Platzhalter durch echte Werte ersetzen):
       AZURE_OPENAI_ENDPOINT    = https://DEIN-RESOURCE.openai.azure.com/
       AZURE_OPENAI_API_KEY     = dein-echter-key
       AZURE_OPENAI_DEPLOYMENT  = gpt-4o   (oder wie dein Deployment heisst)
       AZURE_OPENAI_API_VERSION = 2024-02-01

2. ChromaDB Vektordatenbank aufbauen
   - Die DB/build_db.py muss einmal ausgefuehrt werden, bevor der Server startet
   - Sie liest das PDF ein, zerlegt es in Chunks und speichert die Embeddings
   - Optionen:
     a) Lokal ausfuehren und chroma_db/ Ordner in den Space hochladen, ODER
     b) Ein Startup-Skript im Dockerfile ergaenzen, das build_db.py einmalig ausfuehrt
        (z.B. via ENTRYPOINT-Wrapper der prueft ob /data/chroma_db schon befuellt ist)
   - Der CHROMA_PATH muss auf /data/chroma_db zeigen (HF Persistent Storage)

3. HuggingFace Space erstellen
   - Neuen Space anlegen, Typ: Docker
   - Dieses Repository als Quelle verbinden (oder manuell pushen)
   - Unter Space Settings > Persistent Storage aktivieren (damit die DB erhalten bleibt)
   - Die vier Azure-Secrets aus Schritt 1 eintragen


Kurz-Zusammenfassung des Datenflusses
---------------------------------------
Nutzer-Frage im Browser
  -> /api/chat (FastAPI)
  -> ChromaDB sucht die 5 aehnlichsten Handbuch-Abschnitte
  -> Azure OpenAI bekommt Frage + Kontext und antwortet auf Deutsch
  -> Antwort geht zurueck ans Frontend, wird auch vorgelesen (TTS)
